# 每次学习会话指南

这份文档定义每次说“继续学习”时的执行方式。目标是让学习按完成状态连续推进，而不是按日期硬跳。

## 会话开始

每次开始前先读取：

1. `README.md`
2. `docs/progress.md`
3. `docs/weekly-plan.md`
4. 必要时读取 `docs/code-interpreter-guard-design.md`

## 推进规则

- 以 `docs/progress.md` 的完成状态为准，日期只作为参考。
- 如果上次任务未完成，优先续上，不进入新任务。
- 每次只推进一个最小闭环。
- 周一到周五执行正式学习任务。
- 周六只做可选复盘、补漏、复跑实验和整理问题清单。
- 周日休息，不安排正式学习任务。
- 不擅自扩展到 MVP 外能力。

## 每次输出

每次学习会话应给出：

- 今日目标
- 核心概念解释
- 动手实验步骤
- 系统安全视角
- 验收标准
- 今日记录模板

## 会话结束

学习结束时更新：

- `docs/progress.md`
- `docs/notes/week-XX.md`

如果任务没有完成，记录：

- 当前做到哪里
- 阻塞点
- 下次继续的第一步

## 记录要求

每日笔记简短够用，建议 300-800 字。结构参考 `docs/templates/daily-note.md`，不要求写成文章。
