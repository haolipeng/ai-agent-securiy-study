# 学习笔记

按周分目录，**每完成一天的任务，在对应目录下写一篇简洁技术文档**：

```
docs/notes/
├── week-01/
│   ├── README.md
│   ├── day-01-first-call.md … day-06-tool-args.md
│   └── week-01-summary.md
├── week-02/
│   ├── README.md
│   ├── day-07-read-file.md … day-10-agent-loop.md
│   └── week-02-summary.md
└── ...
```

## 写作要求

- 一篇文档只覆盖一天，聚焦「做了什么、关键代码、实验结论、安全视角」
- 简洁够用，300–800 字，不写成长篇教程
- 代码路径、运行命令、样例输出要可复现
- 依赖安装在项目根目录：`pip install -r requirements.txt`；运行前 `source .venv/bin/activate`
